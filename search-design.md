# 여행지 검색 기능 설계

여행지 검색 기능은 SearchBar 컴포넌트의 '여행지' 버튼을 클릭하면 검색 패널이 열리는 방식으로 동작한다. 패널이 열리면 입력창에 자동으로 포커스가 잡히고, 아무것도 입력하지 않은 상태에서는 서울, 부산, 제주, 강릉, 경주, 여수 등 6개의 추천 여행지가 아이콘과 함께 표시된다.

사용자가 검색어를 입력하면 입력 즉시 전체 여행지 데이터에서 이름이나 설명에 해당 키워드가 포함된 결과를 필터링해서 보여준다. 필터링 결과가 없으면 "검색 결과가 없습니다"라는 메시지를 노출한다. 입력창 오른쪽의 X 버튼을 누르면 검색어가 초기화되고 다시 추천 여행지 목록으로 돌아간다.

키보드 탐색도 지원하는데, ArrowDown을 누르면 다음 항목으로, ArrowUp을 누르면 이전 항목으로 이동한다. 이때 마지막 항목에서 아래를 누르면 첫 번째로, 첫 번째에서 위를 누르면 마지막으로 순환되도록 구현했다. 키보드로 이동 중인 항목은 배경색이 바뀌면서 하이라이트되고, 검색 결과 목록에서는 현재 하이라이트된 항목의 이름이 입력창에도 반영된다. Enter를 누르면 해당 항목이 선택되고, Escape를 누르면 패널이 닫힌다.

컴포넌트 구조는 SearchBar 안에 LocationSearch라는 새 컴포넌트를 만들어서 분리했다. SearchBar에서는 isLocationOpen과 selectedLocation 상태를 관리하고, LocationSearch 내부에서는 query와 highlightIndex 상태를 관리한다. 여행지 필터링은 useMemo를 사용해서 query가 바뀔 때만 재계산되도록 했고, useRef는 입력창 포커스 제어, 패널 외부 클릭 감지, 스크롤 처리에 활용했다. useEffect는 컴포넌트 마운트 시 자동 포커스, 외부 클릭 이벤트 등록/해제, 하이라이트 항목이 보이도록 스크롤하는 용도로 사용했다.

데이터는 popularDestinations와 allDestinations 두 개로 나눠서 관리한다. popularDestinations는 추천 여행지용이고 allDestinations는 검색 필터링 대상이다. 여행지를 선택하면 SearchBar의 여행지 버튼에 선택된 이름이 표시되고 패널이 닫힌다.
