관악bnb 검색 설계 요약

1. 목표

- React에서 fetch API로 Express 검색 서버를 호출한다.
- MongoDB stays collection에 저장된 데이터를 조건에 맞게 조회한다.
- 검색 결과를 같은 페이지 하단에서 즉시 다시 렌더링한다.

2. 필수 검색 조건

- 여행지
- 여행 인원

3. 선택 검색 조건

- 체크인
- 체크아웃
- 반려동물

4. 프론트 구조

App
|- SearchBar
|  |- DestinationField
|  |- DateRangeField
|  |- GuestSelector
|  |- SuggestionList
|- StayGrid

5. 백엔드 구조

server
|- config
|- routes
|- controllers
|- services
|- repositories
|- models
|- seeds
|- scripts

6. 검색 흐름

1) 사용자가 여행지와 인원을 입력한다.
2) SearchBar submit 시 fetch로 /api/stays/search 요청을 보낸다.
3) Controller가 query를 받아 Service로 넘긴다.
4) Service가 검색 규칙을 검증하고 Repository용 query를 만든다.
5) Repository가 MongoDB stays collection을 조회한다.
6) 결과 JSON을 받아 StayGrid가 같은 화면 아래에 다시 렌더링된다.

7. 데이터 모델 핵심 필드

- slug
- title
- summary
- location
- city
- district
- category
- keywords
- maxGuests
- petFriendly
- pricePerNight
- rating
- reviewCount
- availability.startDate
- availability.endDate

8. 실행 순서

- npm install
- cp .env.example .env
- MONGO_URI 입력
- npm run seed
- npm run dev

9. 배포

- Render용 render.yaml 포함
- Web Service 타입으로 배포한다
- 원격 MongoDB URI를 Render 환경변수에 등록하면 된다

10. 검증

- npm run seed 성공
- npm run build 성공
- /api/health 응답 확인
- /api/stays/search?destination=서울&guests=2 응답 확인
